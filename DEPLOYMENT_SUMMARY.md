## 🔐 规则核心功能速记

### 员工权限
```
✅ 能做的：
   • 查看自己的打卡记录
   • 创建自己的打卡记录
   • 修改自己的签退时间
   • 填写工作总结

❌ 不能做的：
   • 查看他人的打卡记录
   • 修改他人的数据
   • 批准加班
   • 修改邀请码
   • 删除任何记录
```

### 管理员权限
```
✅ 能做的：
   • 查看所有员工数据
   • 查看所有打卡记录
   • 批准或拒绝加班
   • 修改邀请码和配置
   • 删除员工账号
   • 查看审计日志

❌ 不能做的：
   • 修改或删除审计日志
     (为了保证记录完整性)
```

## 🆘 遇到问题？

### 最常见的问题

**Q: 规则发布后员工打卡失败**
A: 检查代码中是否正确传入了 `uid` 字段
   → 查看 DEPLOYMENT_STEPS.md 中的 "问题2" 部分

**Q: 管理员无法看到数据**
A: 检查管理员的 role 是否为 'admin'，isActive 是否为 true
   → 查看 DEPLOYMENT_STEPS.md 中的 "问题3" 部分

**Q: 规则发布显示语法错误**
A: 确保复制了完整的代码，没有遗漏任何部分
   → 查看 DEPLOYMENT_STEPS.md 中的 "问题1" 部分

**Q: 需要快速回退**
A: 准备好备份的旧规则，粘贴后重新发布
   → 只需 1-2 分钟就能恢复
