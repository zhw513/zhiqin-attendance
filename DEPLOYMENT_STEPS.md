## 🔧 第六步：故障排除

### 问题1: 发布失败，显示语法错误

**检查清单**:
- [ ] 确保整个文件都复制了
- [ ] 没有遗漏 `}` 或 `;`
- [ ] 没有修改函数名字
- [ ] `appId` 变量名正确

**解决方案**:
```
1. 清空编辑框
2. 重新从文件复制规则
3. 确保复制了所有行（包括最后的 }）
4. 再次粘贴和发布
```

### 问题2: 发布成功，但员工打卡失败

**可能原因**: 代码中 uid 没有正确传递

**检查代码**:
```javascript
// ❌ 错误 - 没有 uid
await addDoc(collection(db, 'artifacts', appId, 'public', 'data', 'attendance'), {
  clockIn: '09:00',
  // 缺少 uid !
});

// ✅ 正确
await addDoc(collection(db, 'artifacts', appId, 'public', 'data', 'attendance'), {
  uid: user.uid,  // ← 必须有
  clockIn: '09:00',
});
```

**修复**:
打开 `src/App.jsx` 搜索 `addDoc(collection(db, 'artifacts'`，确保所有操作都包含正确的 uid

### 问题3: 管理员无法查看数据

**可能原因**: 管理员的 role 不是 'admin' 或 isActive 为 false

**检查**:
```
1. 打开 Firebase Console -> Firestore -> 数据
2. 找到 artifacts -> intelligent-attendance-system-v1 -> public -> data -> users
3. 找到管理员的用户文档
4. 检查字段:
   - role: "admin"  ← 必须是这个
   - isActive: true ← 必须是 true
```

**修复**:
```javascript
// 如果不对，用代码修改（管理员自己可以改）
await updateDoc(doc(db, 'artifacts', appId, 'public', 'data', 'users', adminUid), {
  role: 'admin',
  isActive: true
});
```

### 问题4: 需要快速回退

**如果新规则有问题**:

1. 打开 Firebase Console 规则页面
2. 粘贴备份的旧规则
