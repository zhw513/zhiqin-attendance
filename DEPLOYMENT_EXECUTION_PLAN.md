## 🆘 如果部署失败

### 情况 1: 发布时显示语法错误

**症状**:
```
❌ 语法错误 (第 XX 行)
```

**解决**:
1. 清空编辑框（Ctrl+A + Delete）
2. 重新打开 `FIREBASE_RULES_CLEAN.txt`
3. Ctrl+A 全选
4. Ctrl+C 复制
5. 回到 Firebase，Ctrl+V 粘贴
6. 确保复制了所有内容（包括最后的 `}}`）
7. 再次尝试发布

### 情况 2: 发布成功，但应用出错

**症状**:
```
Uncaught FirebaseError: Missing or insufficient permissions
```

**解决**:
这意味着规则正确生效了，但当前操作没有权限

**检查**:
1. 用错的账号了吗？用员工账号测试员工权限，用管理员账号测试管理员权限
2. 用户状态正确吗？
   - [ ] role: 'admin' 或 'staff'
   - [ ] isActive: true
   - [ ] uid: 正确的 Firebase UID

**修复**:
打开 Firebase Console -> Firestore -> 数据，找到用户文档，检查这些字段

### 情况 3: 部分功能失败

**常见原因**: 代码中没有正确传入 uid

**检查代码**:
在 `src/App.jsx` 中搜索所有数据库操作，确保：
```javascript
✅ 正确: 
await addDoc(collection(...), {
  uid: user.uid,  // ← 必须有
  clockIn: '09:00',
  ...
});

❌ 错误:
await addDoc(collection(...), {
  clockIn: '09:00',  // ← 缺少 uid
  ...
});
```

### 情况 4: 需要快速回退

**如果新规则有严重问题**:

1. 打开备份文件: `FIREBASE_RULES_BACKUP_BEFORE_20260511.txt`
2. Ctrl+A + Ctrl+C 复制
3. 回到 Firebase 规则编辑器
4. Ctrl+A + Delete 清空
5. Ctrl+V 粘贴老规则
6. 点击发布
7. **应用恢复到之前的状态**（只需 1-2 分钟）
