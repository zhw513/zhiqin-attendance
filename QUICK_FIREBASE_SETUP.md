# 🚀 Firebase 安全规则 - 快速设置指南

## ⚡ 5分钟快速替换

### 步骤1️⃣: 复制规则代码
打开文件: `FIREBASE_SECURITY_RULES.firestore`  
**Ctrl+A 全选** -> **Ctrl+C 复制**

### 步骤2️⃣: 打开 Firebase Console
访问: https://console.firebase.google.com/project/intelligent-attendance-s-9289d/firestore/rules

### 步骤3️⃣: 替换规则
1. 清空现有规则（全选后删除）
2. 粘贴新规则（Ctrl+V）
3. 点击 **发布** 按钮（右上角蓝色）

### 步骤4️⃣: 等待完成 ✅
- 规则会在 1-2 秒内生效
- Console 会显示 "发布成功"

## 🎯 规则核心逻辑

### 用户表 (users)
```
员工  → 只读/修改自己   ✅
      → 看不到他人      ✅
      → 不能改权限      ✅

管理员 → 读/写所有人     ✅
       → 可以删除用户   ✅
       → 不能修改日志   ✅
```

### 打卡表 (attendance)
```
员工  → 创建自己打卡    ✅
      → 修改签退/总结   ✅
      → 看不到他人打卡  ✅
      → 不能删除        ✅

管理员 → 看所有打卡     ✅
       → 批准/拒绝     ✅
       → 可以删除      ✅
```

### 配置表 (settings)
```
员工  → 读取配置       ✅
      → 不能修改      ✅

管理员 → 读取配置      ✅
       → 修改配置     ✅
```

### 日志表 (audit_logs)
```
所有人 → 创建日志      ✅
       → 读取自己的   ✅

管理员 → 读取所有     ✅
       → 不能修改    ✅

任何人 → 不能删除     ✅
```

## 🔧 故障排除

### 问题1: "权限被拒绝" 错误
**原因**: 规则生效了  
**检查**:
- [ ] 检查用户 role 是否为 'admin' 或 'staff'
- [ ] 检查用户 isActive 是否为 true
- [ ] 检查 uid 是否匹配

### 问题2: 员工看不到自己的数据
**原因**: uid 不匹配  
**修复**:
```javascript
// 检查 uid 是否正确
console.log('当前用户 uid:', user.uid);
console.log('数据库中的 uid:', attendanceRecord.uid);
```

### 问题3: 管理员权限不生效
**原因**: role 或 isActive 状态不对  
**修复**:
1. 打开 Firebase Console
2. 找到用户文档
3. 检查 `role: 'admin'` 和 `isActive: true`
